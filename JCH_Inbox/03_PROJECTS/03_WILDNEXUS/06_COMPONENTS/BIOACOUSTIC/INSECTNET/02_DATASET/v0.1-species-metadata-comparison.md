## Correction taxonomique

`Metrioptera roeselii` ne retourne aucun resultat sous ce nom strict dans Xeno-canto.

La requete exploitable est :

```text
gen:Roeseliana sp:roeselii
```

Decision :

```text
Utiliser Roeseliana roeselii comme nom de travail Xeno-canto, en gardant Metrioptera roeselii comme synonyme historique.
```

## Especes fortes pour InsectNet V0.1

### 1. Pholidoptera griseoaptera

Point fort :

- 1417 enregistrements ;
- 1335 utilisables pour entrainement interne ;
- 1296 utilisables pour release stricte selon les regles actuelles.

Risque :

- forte concentration Pays-Bas, donc verifier generalisation geographique.

### 2. Tettigonia viridissima

Point fort :

- 631 enregistrements ;
- beaucoup de sample rates eleves ;
- tres bonne espece pilote.

### 3. Leptophyes punctatissima

Point fort :

- 353 enregistrements ;
- excellente proportion open ;
- bon candidat pour release propre.

### 4. Gryllus campestris

Point fort :

- espece connue et naturalistement parlante ;
- 367 enregistrements.

Risque :

- seulement 61 lignes release stricte selon les regles actuelles.

### 5. Roeseliana roeselii

Point fort :

- 268 enregistrements ;
- espece utilisable si nom taxonomique corrige.

Risque :

- proportion review plus elevee.

## Espece a sortir du lot Xeno-canto V0.1

### Cicada orni

Xeno-canto ne retourne aucun resultat avec :

```text
gen:Cicada sp:orni
sp:orni
grp:cicadas cnt:Italy
```

Decision :

```text
Cicada orni reste interessante biologiquement, mais doit etre traitee via BioAcoustica, iNaturalist, ECOSoundSet ou autre source.
```

## Decision V0.1

Construire d'abord InsectNet V0.1 sur 5 especes Xeno-canto exploitables :

1. `Tettigonia viridissima`
2. `Gryllus campestris`
3. `Pholidoptera griseoaptera`
4. `Leptophyes punctatissima`
5. `Roeseliana roeselii` / syn. `Metrioptera roeselii`

Garder `Cicada orni` dans le backlog sources alternatives.

## Prochaine etape

Creer un plan de telechargement audio minimal :

- ne pas telecharger tout le corpus ;
- commencer avec 20 a 50 sons par espece ;
- privilegier `quality_flag=ok` ;
- privilegier `usable_for_public_release=yes` pour eviter les complications ;
- conserver diversite pays et sample rate.
