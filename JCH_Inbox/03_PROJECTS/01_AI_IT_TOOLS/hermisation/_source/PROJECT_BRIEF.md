# Project Brief — PKA [[Hermisation]]

## Objective

Build a durable PKA AI infrastructure that improves:

- continuity between sessions;
- memory of [[decisions]] and workflows;
- procedural reuse;
- light multi-agent orchestration;
- system logging;
- documentary governance;
- productivity without chaos.

## Philosophy

[[Hermisation]] must remain:

- pragmatic;
- progressive;
- controlled;
- documented;
- reversible;
- understandable by JCH.

## What This Is Not

This is not an AGI project, not a replacement for [[Dobby]], and not a mandate to install a complex external framework before the architecture is clear.

## Human Constraints

The system must be step-by-step, inspectable, and maintainable. Stability is more important than sophistication.

## Critical Validation

Human validation is mandatory for:

- deletion;
- mass move;
- global rename;
- external publication;
- exposed API;
- permanent automation;
- purchase;
- sensitive data access.
