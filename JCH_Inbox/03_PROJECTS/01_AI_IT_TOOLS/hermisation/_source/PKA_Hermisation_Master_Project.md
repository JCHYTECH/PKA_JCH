# 1. Mission du projet

Le projet « Hermisation » a pour objectif de transformer progressivement le système PKA_JCH en une infrastructure AI :

- persistante ;
- gouvernée ;
- documentée ;
- modulaire ;
- mémorielle ;
- coordonnée ;
- extensible ;
- stable.

Le projet doit permettre :

- la continuité entre les sessions ;
- la capitalisation des méthodes ;
- l’orchestration légère d’agents ;
- la réduction des pertes de contexte ;
- l’amélioration de la productivité ;
- l’automatisation progressive de workflows validés.

---

# 2. Objectif réel du projet

Le projet NE vise PAS :

- une AGI ;
- une autonomie totale ;
- une architecture ultra complexe ;
- une multiplication incontrôlée des agents ;
- une dépendance à un framework externe ;
- une infrastructure cloud lourde dès le départ.

Le projet vise :

- stabilité ;
- continuité ;
- gouvernance ;
- mémoire ;
- traçabilité ;
- efficacité opérationnelle.

---

# 3. Philosophie générale

## 3.1 Simplicité avant puissance

Toute solution doit être :

- compréhensible ;

## 3.2 Une nouveauté technique à la fois
