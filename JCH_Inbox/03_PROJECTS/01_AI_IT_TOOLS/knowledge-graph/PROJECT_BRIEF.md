# Project Brief — [[Obsidian]] Knowledge Graph

## Objective

Transform PKA_JCH into a relational documentary system that supports:

- human navigation;
- persistent AI memory;
- knowledge graph analysis;
- cross-agent continuity;
- reduced duplication;
- controlled linking between projects, agents, technologies, decisions, suppliers, files, and workflows.

## Operating Principle

The system must remain stable, readable, governable, and reversible. Wikilinking is useful only if it improves continuity and comprehension.

## Scope

Priority entities:

- projects;
- agents;
- companies;
- technologies;
- hardware;
- software;
- standards;
- regulations;
- decisions;
- suppliers;
- clients;
- components;
- workflows;
- procedures;
- APIs;
- protocols.

## Non-Markdown Assets

Non-Markdown files are part of the graph as assets, data, deliverables, or technical components. Markdown remains the explanatory and relational layer.

Supported asset families include PDF, XLSX, CSV, images, JSON, YAML, STEP, STL, scripts, videos, audio, datasheets, and software exports.

## Human Validation Points

JCH validates:

- the first central dictionary;
- important ambiguities;
- conventions;
- any mass rename, move, deletion, or [[archive]] operation.

JCH should not perform manual mass wikilinking.
