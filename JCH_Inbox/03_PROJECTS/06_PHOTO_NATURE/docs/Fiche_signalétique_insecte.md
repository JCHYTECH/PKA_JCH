Bonne question 🙌  

Sur **Squarespace**, tu n’as pas besoin d’un moteur de templates (comme Jekyll ou Eleventy). Le plus simple est :  

1. **Utiliser une page standard** : tu copies le code HTML/CSS de la fiche dans un bloc **Code** de Squarespace.  
   - Les variables `{{ scientific_name }}` etc. devront être remplacées manuellement par tes infos à chaque nouvelle fiche.  
   - Tu peux créer un bloc « modèle » que tu dupliques pour chaque espèce.  

2. **Version conseillée** : celle que tu as déjà dans ton fichier actuel (HTML + CSS simple, avec placeholders `{{ ... }}`) est la plus adaptée à Squarespace.  
   - Pas besoin de YAML ni de moteur externe.  
   - Tu peux remplacer les `{{ ... }}` directement par tes textes et images.  

👉 En résumé :  
- **Squarespace → prends la version “HTML statique avec placeholders”** (celle que tu viens d’éditer).  
- **Jekyll/Eleventy/WordPress → prends la version avec front matter ou variables dynamiques** (l’autre template).
